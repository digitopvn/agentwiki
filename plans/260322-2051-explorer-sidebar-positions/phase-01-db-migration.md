# Phase 1: DB Migration & Schema

## Context
- [Brainstorm report](../reports/brainstorm-260322-2051-explorer-sidebar-positions.md)

- [Shared types](../../packages/shared/src/types/document.ts)

## Overview

Add `position` TEXT column to documents, convert folders `position` from INTEGER to TEXT, create `user_preferences` table. Backfill existing data with sequential fractional positions.

## Key Insights

- Folders already have `position INTEGER DEFAULT 0` — all existing rows have value `0`
- Documents have no position field at all
- D1/SQLite doesn't support `ALTER COLUMN` type change — need workaround (add new column, migrate data, drop old)
- Fractional indexing produces lexicographically sortable strings: `a0` < `a1` < `a2` etc.

## Requirements

### Functional
- Documents must have `position TEXT` column
- Folders must use TEXT-based position (not integer)
- New `user_preferences` table for sort settings
- Existing data backfilled with sequential positions

### Non-functional
- Migration must be idempotent/safe for prod D1
- No data loss during folder position conversion

## Implementation Steps

### 1. Install fractional-indexing

```bash
pnpm -F @agentwiki/api add fractional-indexing
pnpm -F @agentwiki/web add fractional-indexing
```

### 2. Update DB schema (`packages/api/src/db/schema.ts`)

Add `position` to documents table:
```typescript
// In documents table definition, add after `folderId`:
position: text('position').notNull().default('a0'),
```

For folders — SQLite can't alter column type. Strategy: add new `position` TEXT column alongside existing integer. Since all existing folders have `position=0`, we just need to add the text field.

**Simplest approach**: Since D1 supports `ALTER TABLE ADD COLUMN`, add a `position` TEXT column. But we already have an INTEGER `position`.

**Recommended**: Keep current integer `position` column name, add new `positionIndex` TEXT column for fractional indexing. This avoids breaking existing code during migration.

Actually, simpler: since all existing folders have `position=0` and the column is never meaningfully used in the UI yet, we can:
1. Add new column `position_index TEXT DEFAULT 'a0'` to folders
2. Update schema to use `positionIndex` for ordering
3. Update folder-service to use the new column

Add `user_preferences` table:
```typescript
export const userPreferences = sqliteTable('user_preferences', {
  id: text('id').primaryKey(),
  userId: text('user_id').notNull().references(() => users.id),
  tenantId: text('tenant_id').notNull().references(() => tenants.id),
  key: text('key').notNull(),
  value: text('value').notNull(),
  updatedAt: integer('updated_at', { mode: 'timestamp_ms' }).notNull(),
}, (table) => [
  uniqueIndex('idx_user_pref_unique').on(table.userId, table.tenantId, table.key),
])
```

### 3. Generate migration

```bash
cd packages/api && pnpm db:generate
```

Review generated SQL. Should produce:
- `ALTER TABLE documents ADD COLUMN position TEXT NOT NULL DEFAULT 'a0'`
- `ALTER TABLE folders ADD COLUMN position_index TEXT NOT NULL DEFAULT 'a0'`
- `CREATE TABLE user_preferences (...)`

Create a one-time backfill function in `folder-service.ts` or a separate migration script.

### 5. Update shared types

`packages/shared/src/types/document.ts`:
- Add `position: string` to Document interface
- Add `positionIndex: string` to Folder interface (keep `position: number` for backward compat during migration)

`packages/shared/src/schemas/document.ts`:
- Add `reorderItemSchema`:
```typescript
export const reorderItemSchema = z.object({
  type: z.enum(['folder', 'document']),
  id: z.string(),
  parentId: z.string().nullable(),
  position: z.string(), // fractional index
})
```
- Update `updateDocumentSchema` to include `position: z.string().optional()`

### 6. Update folder-service.ts

Change `getFolderTree` to order by `positionIndex` instead of `position`:
```typescript
.orderBy(asc(folders.positionIndex), asc(folders.name))
```

Change `createFolder` to compute initial position (append to end of siblings).

Change `updateFolder` to accept `positionIndex` string.
